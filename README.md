# Vargov® Design — Генератор коммерческих предложений

Приложение автоматически собирает премиальное коммерческое предложение в PDF
(A4, вертикальный) по одному артикулу композиции. Данные о модели берутся
с **vargov.ru** автоматически — пользователь вводит только коммерческие параметры.

Два способа использования: **веб-приложение** (вставить блок данных → получить
PDF) и **командная строка**.

## Установка

```bash
npm install
npx playwright install chromium   # один раз, для живого скрапинга
```

## Веб-приложение

```bash
npm run serve
```

Откройте `http://localhost:8815`, вставьте блок параметров в поле и нажмите
«Сгенерировать PDF» — готовое предложение появится в предпросмотре и доступно
для скачивания. Формат вставляемых данных:

```
Композиция: LC0537
Размер композиции: L8000, W1000 мм
Размер элементов: L400, W250 мм
Количество элементов: 60 шт.
Стоимость композиции: 44 100 USD
Срок производства: 60–90 рабочих дней
Размер и вес груза:
4 м³
200 кг
Стоимость доставки: 6 000 USD
Срок поставки:
до 60 календарных дней
```

Порядок строк не важен; значения могут стоять как после двоеточия, так и на
следующей строке (как «Размер и вес груза» и «Срок поставки» выше). Порт
меняется переменной `PORT`.

Сервер слушает все интерфейсы (`0.0.0.0`), поэтому с телефона в той же Wi-Fi
сети он доступен по адресу `http://<IP-компьютера>:8815` (адрес печатается при
запуске). Для готового PDF на мобильном есть кнопки «Открыть PDF» и «Скачать».

## Облачный хостинг (доступ откуда угодно)

Чтобы генерировать PDF с телефона из любой точки (компьютер выключен), приложение
разворачивается в облаке. Оно **самодостаточно** (данные о 651 композиции
забандлены в `data/`) и работает **без браузера** — страница товара vargov.ru
отдаёт данные обычным HTTPS-запросом, поэтому образ лёгкий.

Есть готовый `Dockerfile` (на `node:20-slim`, `NO_BROWSER=1`) и `render.yaml`.

**Render (проще всего):** New → Blueprint → указать репозиторий и корень
`proposal-generator` → Deploy. Render соберёт Docker-образ и даст постоянный
адрес вида `https://vargov-proposal.onrender.com`. На бесплатном плане сервис
засыпает при простое — первый запрос после паузы идёт ~30 сек.

**Любой хостинг с Docker** (Railway / Fly.io / VPS):

```bash
docker build -t vargov-proposal .
docker run -e PORT=8815 -p 8815:8815 vargov-proposal
```

Переменные окружения: `PORT` (хост задаёт сам), `NO_BROWSER=1` (уже в Dockerfile).

## Командная строка

```bash
npm run proposal
```

Приложение спросит артикул и коммерческие параметры, затем соберёт PDF в
`output/Commercial_Proposal_<SKU>.pdf`.

### Неинтерактивный режим (для скриптов)

Любое поле можно передать флагом; `--no-input` отключает вопросы:

```bash
npm run proposal -- \
  --sku LC0537 \
  --composition-size "L8000, W1000 мм" \
  --element-size "L400, W250 мм" \
  --element-count "60 шт." \
  --price "44 100 USD" \
  --delivery-cost "6 000 USD" \
  --production-time "60–90 рабочих дней" \
  --delivery-time "до 60 календарных дней" \
  --cargo-volume "4 м³" \
  --cargo-weight "200 кг" \
  --no-input
```

`--refresh` — принудительно перескрапить страницу и перекачать изображения,
игнорируя кэш.

## Что собирается автоматически

По артикулу приложение находит страницу товара на vargov.ru (через локальную
базу `../vargov-products.json` как индекс, иначе обходом `sitemap-store.xml`),
затем извлекает: название, характеристики, изображение, размер элемента, ссылку
на 3D-модель, ссылки на видео (YouTube / Vimeo / RuTube). Пустые блоки в PDF
просто пропускаются — «нет данных» никогда не печатается.

Текст «О компании» берётся из фирменного описания Vargov®Design (сверено с
главной страницей vargov.ru) и правится в `templates/proposal.json` → `company`.
Там же — фирменные видеоканалы (`brandVideo`), которые используются, когда у
композиции нет собственного видео.

### Пересчёт в рубли

Стоимость автоматически переводится в рубли по **внутреннему курсу компании**,
опубликованному на главной vargov.ru (виджет «внутренний курс USD»). Курс
читается из HTML сайта, кэшируется на 12 часов (`cache/rate.json`) и при
недоступности сайта берётся из `templates/proposal.json` → `internalRate`.
В коммерческом предложении под каждой суммой в USD показан рублёвый эквивалент,
а под итогом — примечание с текущим курсом.

## Структура PDF

1. **Обложка** — логотип, изображение композиции (первое фото с сайта), артикул, название, дата.
2. **О компании** — описание Vargov®Design (с главной страницы vargov.ru).
3. **Характеристики** — таблица (пустые поля скрыты).
4. **Коммерческое предложение** — суммы в USD и ₽ (внутренний курс), итог, условия оплаты (без НДС).
5. **Ссылки** — QR на страницу товара, сайт, 3D-модель, видео.
6. **Видео** — крупное превью + QR на видео артикула; если у композиции нет
   собственного ролика, ведёт на официальные видеоканалы бренда (YouTube, RuTube).
7. **Контакты** — логотип, каналы связи, шоурум, QR на сайт.

Всё оформление — на русском языке, строго в фирменных цветах Vargov®Design
(чёрное на бумаге + золото `#bf8219`). Дизайн-система описана в [DESIGN.md](DESIGN.md).

## Технологии

- **TypeScript** + `tsx` · **Express** — веб-сервер
- **Прямой HTTPS-запрос** — скрапинг vargov.ru (данные в HTML). **Playwright**
  (Chromium) используется только локально как усиление; в облаке `NO_BROWSER=1`
  отключает его — образ лёгкий, без браузера
- **PDFKit** — генерация PDF с абсолютной вёрсткой (без шаблонов)
- **svg-to-pdfkit** — фирменный логотип рисуется как вектор прямо в PDF
- **sharp** — скачивание и оптимизация фотографии обложки (≤2400px, mozjpeg)
- **qrcode** — QR-коды (рисуются как векторные модули, без растра)

## Векторная графика

Вся брендовая графика и текст — векторные, масштабируются без потери качества:

- **Логотип** — официальный `assets/brand/logo_gold.svg` рендерится в PDF как
  векторные контуры (не растрируется в PNG).
- **QR-коды** — рисуются модулями-прямоугольниками нативно в PDF.
- **Шрифты** — Montserrat встраивается как TTF; текст остаётся настоящим
  (выделяется, копируется) и печатается векторными глифами.
- **Линии, плашки, элементы** — нативные векторные примитивы PDFKit.

Растровой остаётся только фотография композиции на обложке — по определению.

Фирменный акцентный цвет — золото **#bf8219** из логотипа Vargov®Design.

## Кэш

- `cache/<SKU>/product.json` — данные модели
- `cache/<SKU>/originals/` — оригиналы изображений (не качаются повторно)
- `cache/<SKU>/opt/` — оптимизированные для печати JPEG

## Дизайн

Фирменный стиль vargov.ru: чернила по бумаге, единственный золотой акцент
(`#bf8219` — цвет логотипа), шрифт Montserrat, много воздуха. Шрифты и
логотип — из `assets/`. Правки текстов и контактов — в
`templates/proposal.json`.
