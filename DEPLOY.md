# Деплой на Render — постоянная ссылка для телефона

После деплоя приложение будет доступно по адресу вида
`https://vargov-proposal.onrender.com` — **из любой страны, с телефона,
компьютер можно выключить**.

Всё уже подготовлено и проверено: Docker-образ собирается и генерирует PDF,
данные и курс подтягиваются с vargov.ru без браузера, база на 651 товар
встроена как запасной вариант.

---

## Что нужно (бесплатно, аккаунты создаёте только вы)

1. **Аккаунт GitHub** — https://github.com/signup (код нужно где-то хранить).
2. **Аккаунт Render** — https://render.com (можно войти через GitHub; карта не нужна).

> Я не могу создавать аккаунты за вас — это делаете вы. Всё остальное ниже —
> копирование готовых команд.

## Шаг 1. Загрузить код на GitHub

Репозиторий уже собран локально (ветка `main`, первый коммит сделан).
Осталось создать пустой репозиторий на GitHub и отправить туда код.

1. Откройте https://github.com/new
   - **Repository name:** `vargov-proposal`
   - Тип: **Private** (приватный) — рекомендую.
   - **НЕ** ставьте галочки «Add README / .gitignore / license».
   - Нажмите **Create repository**.

2. В терминале в папке `proposal-generator` выполните (подставьте свой логин
   вместо `ВАШ-ЛОГИН`):

   ```bash
   git remote add origin https://github.com/ВАШ-ЛОГИН/vargov-proposal.git
   git push -u origin main
   ```

   При первом `push` откроется окно входа в GitHub — войдите в браузере
   (Git сам сохранит доступ). После этого код окажется на GitHub.

---

## Шаг 2. Развернуть на Render

1. Войдите на https://dashboard.render.com (кнопкой **GitHub**).
2. Нажмите **New +** → **Blueprint**.
3. Выберите репозиторий **vargov-proposal** → **Connect**.
   Render сам найдёт файл `render.yaml` и покажет сервис `vargov-proposal`.
4. Нажмите **Apply** / **Create**.
5. Подождите 3–5 минут, пока идёт первая сборка (в логах будет `npm ci`,
   затем запуск). Когда статус станет **Live** — готово.

Наверху страницы сервиса будет постоянная ссылка, например
`https://vargov-proposal.onrender.com`. Откройте её на телефоне — всё работает.

---

## Как пользоваться

Открываете ссылку на телефоне → вставляете данные композиции →
«Сгенерировать PDF» → «Открыть PDF» / «Скачать». Как на компьютере.

## Важные детали

- **Бесплатный тариф «засыпает»** после ~15 минут простоя. Первый запрос
  после паузы «будит» сервис — страница откроется за ~30–60 секунд, дальше
  быстро. Если нужна работа без пауз — на Render есть платный тариф от $7/мес
  (в `render.yaml` поменять `plan: free` на `plan: starter`).
- **Внутренний курс** подтягивается с главной vargov.ru при каждой генерации
  (кэш 12 часов) — цены в рублях всегда актуальные.
- **Обновить приложение** после правок: выполнить `git push` — Render
  пересоберёт и обновит сервис автоматически (`autoDeploy: true`).

## Проверка Docker-образа локально (необязательно)

Всё уже проверено, но при желании можно убедиться самому:

```bash
docker build -t vargov-proposal .
docker run --rm -p 8899:8815 vargov-proposal
# затем откройте http://localhost:8899
```
